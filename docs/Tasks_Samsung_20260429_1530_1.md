## Decisões de Arquitetura — LOCKED antes da implementação

| ID  | Decisão | Rationale |
|-----|---------|-----------|
| S1  | `X_DIV = 330` | Calibrado em 4 amostras reais; gap confirmado: col esq termina ~315, col dir começa ~351 |
| S2  | Ordem leitura: `for col in ("e", "d")` por página | Mesma lógica do Santander; esquerda sempre antes da direita |
| S3  | Titular extraído do cabeçalho da pág. 1 via regex `Titular\s+([A-Z][A-Z\s]+)` | Formato diferente do Santander; sem concatenação com cartão |
| S4  | Final cartão extraído via regex `Cartão\s+\d{4}\.XXXX\.XXXX\.(\d{4})` | Específico Samsung Itaú |
| S5  | Parcela = penúltimo token quando `RE_PARC.match(tokens[-2])` | Confirmado nas 4 amostras: `NAT*Natura Pag 02/06 53,34` |
| S6  | Seção "Pagamentos efetuados" detectada por label de texto — lançamentos dessa seção = `tipo=Pagamento`, valor positivo no JSON (convenção do projeto) | Seção separada no PDF; não misturar com compras |
| S7  | Seção "Compras parceladas - próximas faturas" → **ignorar completamente** | São lançamentos de faturas futuras, não desta fatura |
| S8  | Validação de total: usar `Lançamentos no cartão` (exclui internacionais) OU `Total dos lançamentos atuais` (inclui internacionais) — preferir `Total dos lançamentos atuais` quando presente | Fatura 02/2026: `Lançamentos no cartão`=915,00 ≠ `Total dos lançamentos atuais`=1.023,24 por causa de internacionais |
| S9  | Lançamentos internacionais (seção `Lançamentos internacionais`) → processar normalmente, valor em R$ (coluna R$), `tipo=Outros` | Detectado na amostra 02/2026 |
| S10 | Tolerância de validação: `R$ 0,10` | Consistente com Santander |
| S11 | `inferir_ano_samsung()` — mesma lógica do Santander (`inferir_ano_parcelado` e `inferir_ano_avista`) | Lógica de retroação é universal; apenas nomeação diferente para clareza |

## TASK-S02: Extração de segmento por coluna

**Arquivo:** `src/cartao_samsung.py`

**Ação:**
1. Implementar `extrair_segmento(page, col: str) -> list` — **idêntico ao Santander**, apenas com `X_DIV = 330`:
   ```python
   def extrair_segmento(page, col: str) -> list:
       words = page.extract_words(keep_blank_chars=False, x_tolerance=3, y_tolerance=3)
       if col == "e":
           words_col = [w for w in words if w["x0"] < X_DIV]
       else:
           words_col = [w for w in words if w["x0"] >= X_DIV]
       linhas_dict = defaultdict(list)
       for w in words_col:
           y = round(w["top"] / 4) * 4
           linhas_dict[y].append(w)
       resultado = []
       ys = sorted(linhas_dict.keys())
       for y in ys:
           tokens = [w["text"] for w in sorted(linhas_dict[y], key=lambda w: w["x0"])]
           resultado.append((y, tokens))
       return resultado
   ```
   > ⚠️ O Santander tem fusão de prefixo solitário ('2' ou '3') — **não replicar** no Samsung; não existe esse padrão nas amostras.

**Critério de aceite:**
- Amostra 02/2026: col esq retorna 16+ linhas com DATA, col dir retorna 12+ linhas com DATA
- Nenhum token da col esq aparece na col dir e vice-versa

   ```python
   KEYWORDS_OUTROS = ["iof", "juros", "multa", "anuidade", "encargo", "refinanc"]

   def classificar_tipo(descricao: str, tem_parcela: bool, secao_atual: str) -> str:
       if secao_atual == "pagamentos":
           return "Pagamento"
       dl = descricao.lower()

   ```

- `NAT*Natura Pag 02/06` com `tem_parcela=True` → `"Compra parcelada"`

## TASK-S05: Validação de total

**Arquivo:** `src/cartao_samsung.py`

**Ação:**
1. Implementar `validar_total(lancamentos: list, texto: str) -> tuple[bool, float, float]`:
   ```python
   def validar_total(lancamentos: list, texto: str):
       # Preferir "Total dos lançamentos atuais" (inclui internacionais)
       m = re.search(r"Total dos lançamentos atuais\s+([\d.]+,\d{2})", texto)
       if not m:
           # Fallback: "Lançamentos no cartão"
           m = re.search(r"Lançamentos no cartão\s+([\d.]+,\d{2})", texto)
       if not m:
           return False, 0.0, 0.0
       total_pdf  = float(m.group(1).replace(".", "").replace(",", "."))
       tipos_debito = {"Compra parcelada", "Compra à vista", "Outros"}
       total_calc = sum(abs(l["valor"]) for l in lancamentos if l["tipo"] in tipos_debito)
       return abs(total_pdf - total_calc) < 0.10, total_pdf, total_calc
   ```
   > ⚠️ Pagamentos não entram no total de validação — são créditos, não débitos.

**Critério de aceite:**
- Amostra 01/2026: total_pdf=1139,85; diff < 0,10 → OK
- Amostra 02/2026: total_pdf=1023,24 (inclui internacionais); diff < 0,10 → OK
- Amostra 03/2026: total_pdf=1396,74; diff < 0,10 → OK
- Amostra 04/2026: total_pdf=1050,75; diff < 0,10 → OK
