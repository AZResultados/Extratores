## TASK-05: Integração VBA Exec e Fail-Fast (Excel) [BR-03]

**Alvo:** `Extratores.xlsm` (Módulo `ModComum`)

**Ação:**
1. Substituir `.Run` por `oExec = WScript.Shell.Exec(...)`.
2. Capturar buffers **obrigatoriamente nesta ordem** (evita deadlock de pipe):
   ```vba
   jsonStr = oExec.StdOut.ReadAll   ' lê stdout PRIMEIRO
   errStr  = oExec.StdErr.ReadAll   ' lê stderr DEPOIS
   ```
3. Avaliação de saída:
   - `oExec.ExitCode <> 0` → exibir `errStr` via `MsgBox`, descartar `jsonStr`, `Exit Sub`.
   - `oExec.ExitCode = 0` e `Len(Trim(errStr)) > 0` → exibir `MsgBox "Aviso técnico: " & errStr` (não abortar).
   - `oExec.ExitCode = 0` → fazer parse do `jsonStr`, checar campo `avisos`, exibir `MsgBox` se não vazio, prosseguir para gravação.
4. Parsing do `jsonStr`: implementar via `MSScriptControl.ScriptControl` ou rotina de conversão leve nativa VBA.

**Critério de aceite:**
- `C:\Temp\extratores_output.json` nunca criado durante execução.
- Falha no Python (ExitCode 1) → Excel não grava nada e exibe erro.
- Aviso no `errStr` com ExitCode 0 → exibido sem abortar gravação.

> ⚠️ **Risco deadlock documentado:** `Exec()` bloqueia se JSON exceder buffer do pipe (~4 KB Windows). Mitigação: `StdOut.ReadAll` obrigatoriamente **antes** de verificar `ExitCode` — conforme ordem acima.

## TASK-07: Módulo Compartilhado de Descriptografia (Python)

**Alvo:** `src/pdf_decrypt.py` (novo), `src/cartao_mercadopago.py`, `src/cartao_santander.py`

**Contexto:** Com múltiplos extratores previstos no projeto, duplicar lógica de descriptografia em cada um gera inconsistência e retrabalho. A solução é extrair para um módulo único importado por todos os extratores que precisem tratar PDF com senha.

**Ação:**
1. Criar `src/pdf_decrypt.py` com função `descriptografar(pdf_path: Path, password: str)`:
   - Se `password` vazio → retorna o `Path` original sem modificação.
   - Se `password` preenchido → abre com `pikepdf`, salva em `io.BytesIO`, retorna o buffer in-memory.
   - Falha (senha incorreta, arquivo corrompido) → `sys.exit(1)` + mensagem descritiva em `sys.stderr`.
2. Em `cartao_mercadopago.py` e `cartao_santander.py`:
   - Adicionar `from pdf_decrypt import descriptografar`.
   - Remover qualquer lógica local de descriptografia.
   - Chamar `source = descriptografar(pdf_path, args.password)` e passar `source` diretamente para `pdfplumber.open(source)`.

**Interface do módulo:**
```python
# src/pdf_decrypt.py
import io
from pathlib import Path

def descriptografar(pdf_path: Path, password: str):
    """Retorna BytesIO (descriptografado in-memory) ou o Path original se sem senha."""
    if not password:
        return pdf_path
    import pikepdf
    with pikepdf.open(pdf_path, password=password) as pdf:
        buf = io.BytesIO()
        pdf.save(buf)
        buf.seek(0)
    return buf
```

**Critério de aceite:**
- `src/pdf_decrypt.py` existe e é importável por qualquer extrator do projeto.
- Extratores existentes (Mercado Pago, Santander) não contêm lógica de descriptografia própria — delegam inteiramente ao módulo.
- Novos extratores ganham suporte a PDF com senha adicionando apenas `from pdf_decrypt import descriptografar` + uma linha de chamada.
- Zero arquivos PDF descriptografados em disco (invariante mantida do BR-06).
- Erro de senha: `ExitCode = 1`, mensagem legível no `stderr`.
- `--password` permanece em todos os extratores por simetria de CLI — o módulo é transparente para o VBA.

> ⚠️ **Escopo do módulo:** `pdf_decrypt.py` trata exclusivamente abertura/descriptografia. Parse de conteúdo, regex de lançamentos e envelope JSON permanecem em cada extrator.

## Decisões de Arquitetura Incorporadas nesta Versão

| ID | Decisão | Rationale |
|----|---------|-----------|
| A1 | `id_lote` formato `{EMISSOR}-{YYYYMMDD}-{HHMMSS}` | Auditável por humano (BR-04) |
| A2 | Modo escrita APPEND + rollback por ID_Lote | Histórico preservado; reversão granular (BR-04) |
| A3 | Regex de parcela por emissor, não genérica | Evita falsos positivos; testada por PDF real |
| A4 | `StdOut.ReadAll` antes de `ExitCode` | Previne deadlock de pipe (TASK-05) |
| A5 | `errStr` não-vazio com ExitCode=0 → aviso, não abort | Warnings técnicos visíveis sem bloquear operação |
| A6 | `sys.exit(2)` para argumento ausente explicitado | Falha rápida e clara para o operador |
| A7 | Descriptografia extraída para `pdf_decrypt.py` compartilhado | Evita duplicação entre extratores; novos emissores ganham suporte a senha com uma importação |
