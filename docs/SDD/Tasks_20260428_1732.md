## TASK-10: Banco de Senhas SQLite — db_senha [BR-08]

**Status:** ✅ CONCLUÍDA — 2026-04-28 (schema v2 com identificador)

**Schema atual (v2):**
```sql
CREATE TABLE clientes_senhas (
    cliente       TEXT NOT NULL,
    emissor       TEXT NOT NULL,
    identificador TEXT NOT NULL,   -- BU/filial/titular — '' para cliente com senha única
    senha         TEXT NOT NULL,
    PRIMARY KEY (cliente, emissor, identificador)
)
```

**Campo `identificador`:**
- Permite múltiplas senhas por `(cliente, emissor)` — um por BU/filial/titular
- `''` (vazio) para clientes com senha única por emissor
- Sem `cpf_cnpj` — irrelevante para o mecanismo de abertura

**Funções disponíveis em `db_senha.py`:**

| Função | Uso |
|--------|-----|
| `get_senha(cliente, emissor, identificador)` | Lookup direto |
| `get_todas_senhas(cliente)` | Router: cicla todas as senhas do cliente |
| `set_senha(cliente, emissor, identificador, senha)` | Upsert |
| `remover_senha(cliente, emissor, identificador)` | Delete |
| `listar()` | Administração — sem expor senhas |

**Migração automática** de schema v1 (sem `identificador`) para v2 na primeira execução.

**Setup pelo operador:**
```bash
# Uma BU por cliente
python src/setup_senha.py add CLIENTE-A santander filial-sp

# Múltiplas BUs
python src/setup_senha.py add CLIENTE-A santander filial-rj

python src/setup_senha.py list
```

## TASK-11: Router de Emissor + Entry Point Único [NFR-01 / NFR-03]

**Alvo:** `src/pdf_router.py` (novo), `src/extrator.py` (novo), `src/cartao_mercadopago.py`, `src/cartao_santander.py`, `vba/ModComum.bas`, `vba/ModMP.bas`, `vba/ModSantander.bas`

**Status:** ✅ CONCLUÍDA — 2026-04-28

**Contexto:** O emissor de um PDF não pode ser determinado antes de abrir o arquivo. Usar o sucesso de senha como critério de roteamento é incorreto — PDFs de emissores diferentes podem ter a mesma senha. A solução é um módulo de roteamento que abre o PDF e identifica o emissor por fingerprint de conteúdo, independente da senha.

**Arquitetura implementada:**

```
extrator.py (entry point único)
    └── pdf_router.rotear(pdf_path, cliente)
         ├── tenta abrir sem senha
         │    └── se abriu: detectar_emissor(texto) via fingerprint
         └── cicla get_todas_senhas(cliente)
              └── tentar_descriptografar(pdf_path, senha)
                   └── se abriu: detectar_emissor(texto) via fingerprint
                        └── retorna (emissor, source, identificador)
    └── EXTRATORES[emissor](pdf_path, source)
         ├── cartao_mercadopago.processar_arquivo()
         └── cartao_santander.processar_arquivo()
```

**Princípios mantidos:**
- Extratores são módulos independentes — layout de um não afeta o outro
- Adicionar novo emissor = criar `cartao_novo.py` + fingerprint em `pdf_router.py` + entrada em `extrator.EXTRATORES`
- VBA não muda para novos emissores

**Arquivos criados:**

`src/pdf_router.py`
- `rotear(pdf_path, cliente)` → `(emissor, source, identificador)`
- `detectar_emissor(texto)` — fingerprints por emissor
- `tentar_descriptografar()` — variante não-fatal para ciclar senhas
- Senha ≠ emissor: identificação sempre por conteúdo, nunca por qual senha abriu

`src/extrator.py`
- Entry point único: `python extrator.py --input-dir X --cliente Y`
- Sem `--password` — router obtém senhas do banco automaticamente
- `id_lote` com prefixo do emissor se lote puro (`MP-`, `SA-`), `EXT-` se misto
- Adiciona campo `identificador` em cada lançamento para rastreabilidade de BU

**Arquivos modificados:**

`src/pdf_decrypt.py`
- `tentar_descriptografar()` adicionada — levanta Exception (uso exclusivo do router)
- `descriptografar()` mantida — fatal, uso nos extratores standalone

`src/cartao_mercadopago.py` e `src/cartao_santander.py`
- `processar_arquivo(pdf_path, source)` adicionada — chamada pelo `extrator.py`
- `processar_pasta()` refatorada para usar `processar_arquivo()`
- `__main__` mantido para testes standalone

`vba/ModComum.bas`
- Assinatura simplificada: sem `celulaScript` (script único em Config B2)
- Config B2 = `extrator.py` para todos os emissores
- Sem parâmetro `senha` — Python gerencia internamente

`vba/ModMP.bas` e `vba/ModSantander.bas`
- Chamam `ProcessarExtrator` sem referência a script específico por emissor
- Diferem apenas no `inputDir` (B5 vs B6)

**Para adicionar novo emissor no futuro:**
1. Criar `src/cartao_novo.py` com `processar_arquivo(pdf_path, source)`
2. Adicionar fingerprint em `pdf_router.FINGERPRINTS`
3. Adicionar entrada em `extrator.EXTRATORES`
4. Nenhuma alteração no VBA

## Decisões de Arquitetura Incorporadas nesta Versão

| ID | Decisão | Rationale |
|----|---------|-----------|
| A1 | `id_lote` formato `{EMISSOR}-{YYYYMMDD}-{HHMMSS}` | Auditável por humano (BR-04) |
| A2 | Modo escrita APPEND + rollback por ID_Lote | Histórico preservado; reversão granular |
| A3 | Regex de parcela por emissor, não genérica | Evita falsos positivos |
| A4 | `StdOut.ReadAll` antes de `ExitCode` | Previne deadlock de pipe |
| A5 | `errStr` não-vazio com ExitCode=0 → aviso, não abort | Warnings visíveis sem bloquear |
| A6 | `sys.exit(2)` para argumento ausente | Falha rápida e clara |
| A7 | `pdf_decrypt.py` compartilhado | Novos emissores com senha: uma importação |
| A8 | ScriptControl substitui parser JSON manual | VBA como acionador puro (NFR-01) |
| A9 | `sys.stdout.reconfigure()` | Cross-platform, API nativa Python 3.7+ |
| A10 | Banco SQLite — chave `(cliente, emissor, identificador)` | Multi-BU; mesmo código para SQLite e MySQL |
| A11 | Cadeia de prioridade: `db_senha` > stdin > `--password` | MVP seguro; Docker sem mudança de código |
| A12 | Identificação de emissor por fingerprint, não por senha | Senha abre o PDF; conteúdo identifica o emissor |
| A13 | Entry point único `extrator.py` + extratores independentes | Novos emissores sem tocar VBA; isolamento de layout |
