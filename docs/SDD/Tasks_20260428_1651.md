## TASK-10: Credenciais no Windows Credential Manager (keyring) [BR-08]

**Alvo:** `requirements.txt`, `src/cartao_santander.py`, `src/cartao_mercadopago.py`, `vba/ModSantander.bas`, `vba/ModComum.bas`

**Status:** ✅ CONCLUÍDA — 2026-04-28

**Contexto:** Senhas armazenadas na aba Senhas do Excel (texto claro) eram o último vetor de exposição de credenciais. Com keyring, a senha fica criptografada no Windows Credential Manager, vinculada à conta do usuário — sem arquivo de texto, sem planilha, sem argumento CLI visível.

**Ações executadas:**

**`requirements.txt`**
- Adicionado `keyring==25.6.0`

**`src/setup_senha.py`** (novo)
- Utilitário de configuração único para o operador:
  ```bash
  python src/setup_senha.py santander   # registra senha
  python src/setup_senha.py --remover santander  # remove senha
  ```
- Usa `getpass` — senha digitada sem aparecer na tela

**`src/cartao_santander.py` e `src/cartao_mercadopago.py`**
- Cadeia de prioridade de senha implementada:
  ```python
  # 1. --password (debug)
  password = args.password
  # 2. keyring (MVP Windows — sobrescreve --password)
  stored = keyring.get_password("extratores", "santander")
  if stored:
      password = stored
  # 3. stdin (Docker/API — prioridade máxima)
  if not sys.stdin.isatty():
      linha = sys.stdin.readline().strip()
      if linha:
          password = linha
  ```

**`vba/ModSantander.bas`**
- Removida leitura da aba Senhas (`wsSenhas`)
- Removido parâmetro senha da chamada a `ProcessarExtrator`

**`vba/ModComum.bas`**
- Removido parâmetro `Optional senha As String = ""` da assinatura
- Removido bloco stdin de entrega de senha
- Substituído por `oExec.StdIn.Close` comentado

**Critério de aceite:**
- Aba Senhas do Excel pode ser deletada
- Senha nunca aparece em linha de comando, variável de ambiente ou planilha
- `python src/setup_senha.py santander` registra a senha no Credential Manager
- Extrator Santander funciona sem nenhum argumento de senha no VBA
- Em Docker: senha injetada via stdin pelo orquestrador (prioridade máxima)

**Setup pelo operador (uma vez):**
```bash
cd C:\Dev\projetos\Extratores
venv\Scripts\activate
pip install keyring
python src/setup_senha.py santander
```

## Decisões de Arquitetura Incorporadas nesta Versão

| ID | Decisão | Rationale |
|----|---------|-----------|
| A1 | `id_lote` formato `{EMISSOR}-{YYYYMMDD}-{HHMMSS}` | Auditável por humano (BR-04) |
| A2 | Modo escrita APPEND + rollback por ID_Lote | Histórico preservado; reversão granular (BR-04) |
| A3 | Regex de parcela por emissor, não genérica | Evita falsos positivos; testada por PDF real |
| A4 | `StdOut.ReadAll` antes de `ExitCode` | Previne deadlock de pipe (TASK-05) |
| A5 | `errStr` não-vazio com ExitCode=0 → aviso, não abort | Warnings técnicos visíveis sem bloquear operação |
| A6 | `sys.exit(2)` para argumento ausente | Falha rápida e clara para o operador |
| A7 | Descriptografia centralizada em `pdf_decrypt.py` | Evita duplicação; novos emissores com uma importação |
| A8 | ScriptControl substitui parser JSON manual no VBA | VBA como acionador puro (NFR-01); elimina locale |
| A9 | `sys.stdout.reconfigure()` em vez de `TextIOWrapper` | API nativa Python 3.7+; cross-platform |
| A10 | Credenciais no Windows Credential Manager via keyring | Zero texto claro; VBA não toca senhas (BR-08) |
| A11 | Cadeia de prioridade: stdin > keyring > `--password` | MVP seguro; Docker/API sem mudança de código |
