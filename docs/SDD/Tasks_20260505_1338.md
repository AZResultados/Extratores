## TASK-13: ModConfig — Caminhos em Módulo VBA [NFR-01]

**Status:** ✅ CONCLUÍDA — 2026-04-28

**Contexto:** Caminhos de executáveis e scripts estavam na aba Config do Excel (células B1–B4). Isso cria dependência de planilha, dispersa configuração e viola o princípio de manter o Excel limpo. A aba Config pode agora ser removida inteiramente.

**Implementação:**

`vba/ModConfig.bas` (novo):
```vba
Private Const BASE_DIR As String = "C:\Dev\projetos\Extratores"

Public Function PythonExe() As String
    PythonExe = BASE_DIR & "\venv\Scripts\python.exe"
End Function
Public Function ExtratorScript() As String
    ExtratorScript = BASE_DIR & "\src\extrator.py"
End Function
Public Function SetupSenhaScript() As String
    SetupSenhaScript = BASE_DIR & "\src\setup_senha.py"
End Function
Public Function SetupClienteScript() As String
    SetupClienteScript = BASE_DIR & "\src\setup_cliente.py"
End Function
```

**Para mudar de máquina ou pasta:** alterar apenas `BASE_DIR` em `ModConfig.bas`.

## TASK-14: Logging Estruturado — ✅ CONCLUÍDA — 2026-04-29

**Contexto:** Sem observabilidade em produção — falhas silenciosas, impossível auditoria de processamento.

**Implementação:**

`src/logger.py` (novo):
- `RotatingFileHandler` → `~/.extratores/extrator.log` (5 MB, 3 backups)
- Formato: `%(asctime)s | %(levelname)-8s | %(name)s | %(message)s`
- Nível padrão: INFO (override via env `EXTRATORES_LOG_LEVEL`)
- Fallback para stderr se o diretório não puder ser criado
- `_setup()` idempotente — safe para múltiplos imports

**Módulos integrados:**

| Módulo | Logger | Eventos |
|--------|--------|---------|
| `extrator.py` | `extratores.extrator` | início/lote/PDF processado/erro |
| `pdf_router.py` | `extratores.router` | emissor detectado/aviso senha |
| `cartao_santander.py` | `extratores.santander` | início/OK/divergência/titular |
| `cartao_mercadopago.py` | `extratores.mercadopago` | início/OK/divergência/cartão |
| `db_senha.py` | `extratores.db` | CRUD senhas (sem expor o valor) |
| `db_cliente.py` | `extratores.db` | CRUD clientes |

**Restrições:** senhas, CPFs e números completos de cartão nunca são logados. Falha no logging não causa `sys.exit(1)`.

## TASK-16: Parser JSON Puro VBA substitui ScriptControl — ✅ CONCLUÍDA — 2026-04-29

**Contexto:** `MSScriptControl` é um componente COM 32-bit Windows-only, ausente em sistemas 64-bit puros e em qualquer ambiente não-Windows. Causava erros de runtime em algumas configurações do Excel 365. A dependência foi eliminada integralmente.

**Motivação adicional:** `ScriptControl` executava JavaScript arbitrário (`ExecuteStatement`) sobre o JSON recebido do Python — vetor de injeção de código caso o JSON fosse adulterado. O parser VBA puro não tem essa superfície de ataque.

**Implementação — `vba/ModComum.bas`:**

| Item | Antes | Depois |
|------|-------|--------|
| Parse do JSON | `MSScriptControl.ScriptControl` / JScript | `ParseExtratorJSON()` — VBA puro |
| Stdout | `print(json.dumps(envelope, ensure_ascii=False))` | `sys.stdout.write(json.dumps(envelope, ensure_ascii=True))` |
| Leitura stdout | `oExec.StdOut.ReadAll` | `Trim(oExec.StdOut.ReadAll)` |
| Escape de parâmetros CLI | `\"` (backslash) | `""` (convenção cmd.exe) |
| Nomes de variáveis | `inStr`, `val` (conflito com builtins VBA) | `dentroStr`, `sVal` |

**Funções adicionadas:**

- `ParseExtratorJSON(jsonStr) As Collection` — localiza array `[…]`, extrai objetos `{…}` de nível 1 via parser char-a-char; array `[]` vazio → Collection vazia, sem erro
- `ParseJSONObject(objStr) As Object` — preenche `Scripting.Dictionary` com pares chave/valor do objeto
- `UnescapeJSON(s) As String` — trata `\"`, `\\`, `\/`, `\n`, `\r`, `\t` e `\uXXXX` (necessário pois `ensure_ascii=True` converte ã→`ã`, ç→`ç`, etc.)
- `ConvertJSONValue(key, sVal, isStr) As Variant` — converte tipos: `valor`→Double via `Val()` (locale-safe); `parcela_num`/`qtde_parcelas`→Long; `data_compra`/`vencimento`→Date via `DateSerial` (locale-safe); demais→String

**Dependências eliminadas:** `MSScriptControl`, `JScript`, `sc.Eval`, `sc.ExecuteStatement`

## TASK-17: Extrator Nubank RDB — ✅ CONCLUÍDA — 2026-05-05

**Alvo:** `src/extrator_nubank_rdb.py` + `src/pdf_router.py`

**Implementação:**

`src/extrator_nubank_rdb.py` (novo):
- Parser linha a linha com máquina de estado para movimentações multilinhas ("Compra por" / "Rendimento até")
- Lançamentos gerados conforme Seção 7 do Design Doc: Resgate RDB (saldo líquido), IR s/ Resgate RDB, IOF s/ Resgate RDB, Aplicação RDB, Rendimento RDB
- Validação per-row: `valor_bruto − IR − IOF = saldo_líquido` (tolerância R$ 0,10)
- Argumento opcional `--saldo-abertura`: valida `abertura − resgates_bruto + aplicações + rendimento = saldo_final_pdf` (tolerância R$ 0,10)
- `processar_arquivo(pdf_path, source, saldo_abertura=None)` integrado em `extrator.py`
- Logger `extratores.nubank_rdb`; fail-fast sem try/except silencioso

`src/pdf_router.py`:
- Fingerprint adicionado: `"nubank_rdb": ["RDB Resgate Imediato", "Caixinhas PJ"]`

`src/extrator.py`:
- `import extrator_nubank_rdb as nrdb`
- `EXTRATORES["nubank_rdb"] = nrdb.processar_arquivo`
- `PREFIXOS_LOTE["nubank_rdb"] = "NRD"`

**Validação com PDF abril/2026:**
- 15 lançamentos: 7 Resgate RDB + 6 IR s/ Resgate RDB + 1 Aplicação RDB + 1 Rendimento RDB
- Saldo de abertura R$ 157.611,68 → saldo final calculado R$ 116.917,99 = saldo_final_pdf ✅
