## TASK-10: Banco de Senhas SQLite — db_senha (MVP → Docker) [BR-08]

**Alvo:** `src/db_senha.py` (novo), `src/setup_senha.py` (reescrito), `src/cartao_santander.py`, `src/cartao_mercadopago.py`, `requirements.txt`

**Status:** ✅ CONCLUÍDA — 2026-04-28

**Contexto:** keyring armazena uma senha por emissor — não escala para múltiplos clientes com CPF/CNPJ diferentes. SQLite resolve nativamente com a chave `(cliente, emissor)` e já é a arquitetura alvo de produção (MySQL via `EXTRATORES_DB` env var).

**Schema do banco:**
```sql
CREATE TABLE clientes_senhas (
    cliente  TEXT NOT NULL,
    emissor  TEXT NOT NULL,
    cpf_cnpj TEXT,            -- metadado: documenta origem da senha
    senha    TEXT NOT NULL,   -- dígitos efetivos usados na abertura do PDF
    PRIMARY KEY (cliente, emissor)
)
```

**Arquivos criados/alterados:**

`src/db_senha.py` (novo — stdlib pura, sem dependências)
- `get_senha(cliente, emissor)` → retorna senha ou `""` se não cadastrada
- `set_senha(cliente, emissor, senha, cpf_cnpj)` → upsert
- `remover_senha(cliente, emissor)` → delete
- `listar()` → lista cadastros sem expor senhas
- Banco em `~/.extratores/dados.db` (fora do repo)
- Override via `EXTRATORES_DB` env var (Docker: aponta para MySQL)

`src/setup_senha.py` (reescrito)
```bash
python src/setup_senha.py add    CLIENTE-A santander   # cadastra/atualiza
python src/setup_senha.py remove CLIENTE-A santander   # remove
python src/setup_senha.py list                         # lista sem expor senhas
```

`src/cartao_santander.py` e `src/cartao_mercadopago.py`
- Cadeia de prioridade implementada:
  ```python
  password = args.password                           # 1. debug
  stored = get_senha(args.cliente, "santander")      # 2. banco (MVP/produção)
  if stored: password = stored
  if not sys.stdin.isatty():                         # 3. stdin (Docker/API)
      linha = sys.stdin.readline().strip()
      if linha: password = linha
  ```

`requirements.txt`
- `keyring` removido — `sqlite3` é stdlib

**Critério de aceite:**
- Aba Senhas do Excel pode ser deletada
- Senha nunca aparece em CLI, env var ou planilha
- Setup: `python src/setup_senha.py add CLIENTE santander`
- Múltiplos clientes com senhas diferentes para o mesmo emissor: ✅
- Docker: `EXTRATORES_DB=mysql://...` → mesma interface, banco diferente

**Path de migração Docker:**
- MVP: `~/.extratores/dados.db` (SQLite)
- Produção: `EXTRATORES_DB=mysql+pymysql://user:pass@host/db` (requer abstração ORM — pendente pós-MVP)

## Decisões de Arquitetura Incorporadas nesta Versão

| ID | Decisão | Rationale |
|----|---------|-----------|
| A1 | `id_lote` formato `{EMISSOR}-{YYYYMMDD}-{HHMMSS}` | Auditável por humano (BR-04) |
| A2 | Modo escrita APPEND + rollback por ID_Lote | Histórico preservado; reversão granular (BR-04) |
| A3 | Regex de parcela por emissor, não genérica | Evita falsos positivos; testada por PDF real |
| A4 | `StdOut.ReadAll` antes de `ExitCode` | Previne deadlock de pipe (TASK-05) |
| A5 | `errStr` não-vazio com ExitCode=0 → aviso, não abort | Warnings técnicos visíveis sem bloquear operação |
| A6 | `sys.exit(2)` para argumento ausente | Falha rápida e clara para o operador |
| A7 | Descriptografia centralizada em `pdf_decrypt.py` | Evita duplicação; novos emissores com uma importação |
| A8 | ScriptControl substitui parser JSON manual no VBA | VBA como acionador puro (NFR-01); elimina locale |
| A9 | `sys.stdout.reconfigure()` em vez de `TextIOWrapper` | API nativa Python 3.7+; cross-platform |
| A10 | Banco de senhas SQLite — chave `(cliente, emissor)` | Multi-cliente; mesmo código para SQLite e MySQL |
| A11 | Cadeia de prioridade: `db_senha` > stdin > `--password` | MVP seguro; Docker/API sem mudança de código |
