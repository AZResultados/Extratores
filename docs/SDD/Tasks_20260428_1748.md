## TASK-10: Banco de Senhas SQLite — Schema Simplificado [BR-08]

**Status:** ✅ CONCLUÍDA — 2026-04-28

**Schema final:**
```sql
CREATE TABLE senhas_pdf (
    cliente TEXT NOT NULL,
    senha   TEXT NOT NULL,
    PRIMARY KEY (cliente, senha)
)
```

**Decisão de design:** emissor e identificador não são armazenados. O router detecta o emissor pelo conteúdo do PDF após abrir — senha serve apenas para abrir o arquivo. A deduplicação é nativa via `PRIMARY KEY`.

**API `db_senha.py`:**

| Função | Comportamento |
|--------|--------------|
| `set_senha(cliente, senha)` | Retorna `True` se nova, `False` se duplicata (ignorada) |
| `get_todas_senhas(cliente)` | Lista de senhas para o router ciclar |
| `remover_senha(cliente, senha)` | Remove entrada exata |
| `listar()` | `[(cliente, total_senhas)]` — sem expor senhas |

**Setup pelo operador:**
```bash
# Terminal
python src/setup_senha.py add CLIENTE-A
# digita senha sem aparecer na tela

# Ou via botão no Excel (ModSenhas.CadastrarSenhaPDF)
```

## TASK-11: Router de Emissor + Entry Point Único + Cadastro VBA [NFR-01 / NFR-03]

**Status:** ✅ CONCLUÍDA — 2026-04-28

**Arquitetura:**
```
extrator.py  (entry point único — Config B2)
    └── pdf_router.rotear(pdf_path, cliente)
         ├── tenta abrir sem senha → fingerprint → emissor
         └── cicla get_todas_senhas(cliente)
              └── tentar_descriptografar → fingerprint → emissor
         └── retorna (emissor, source)
    └── EXTRATORES[emissor](pdf_path, source)
         ├── cartao_mercadopago.processar_arquivo()
         └── cartao_santander.processar_arquivo()
```

**Princípios:**
- Extratores são módulos independentes — layout de um não afeta o outro
- Senha ≠ identificação de emissor: fingerprint de conteúdo é sempre o critério
- VBA não muda para novos emissores

**Para adicionar novo emissor:**
1. Criar `src/cartao_novo.py` com `processar_arquivo(pdf_path, source)`
2. Adicionar fingerprint em `pdf_router.FINGERPRINTS`
3. Adicionar entrada em `extrator.EXTRATORES`
4. Zero mudanças no VBA

**Novo `vba/ModSenhas.bas`:**
- `CadastrarSenhaPDF()` — botão para cadastrar nova senha
- `RemoverSenhaPDF()` — botão para remover senha
- Senha sempre via stdin (não aparece em ps aux)
- Feedback ao operador: "Senha cadastrada" ou "Senha já existente"

## Decisões de Arquitetura Incorporadas nesta Versão

| ID | Decisão | Rationale |
|----|---------|-----------|
| A1 | `id_lote` formato `{EMISSOR}-{YYYYMMDD}-{HHMMSS}` | Auditável por humano (BR-04) |
| A2 | Modo escrita APPEND + rollback por ID_Lote | Histórico preservado |
| A3 | Regex de parcela por emissor, não genérica | Evita falsos positivos |
| A4 | `StdOut.ReadAll` antes de `ExitCode` | Previne deadlock de pipe |
| A5 | `errStr` não-vazio com ExitCode=0 → aviso, não abort | Warnings visíveis sem bloquear |
| A6 | `sys.exit(2)` para argumento ausente | Falha rápida e clara |
| A7 | `pdf_decrypt.py` compartilhado | Novos emissores com senha: uma importação |
| A8 | ScriptControl substitui parser JSON manual | VBA como acionador puro (NFR-01) |
| A9 | `sys.stdout.reconfigure()` | Cross-platform, API nativa Python 3.7+ |
| A10 | Schema `(cliente, senha)` com PRIMARY KEY | Dedup nativo; sem metadados desnecessários |
| A11 | Cadeia de prioridade: `db_senha` > stdin > `--password` | MVP seguro; Docker sem mudança |
| A12 | Emissor identificado por fingerprint, nunca por senha | Senhas podem ser iguais entre bancos |
| A13 | Entry point único + extratores independentes | Novos emissores sem tocar VBA |
| A14 | Senha via stdin no cadastro VBA | Nunca aparece em linha de comando |
