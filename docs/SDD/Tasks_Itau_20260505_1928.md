## TASK-IT-02: Implementar `src/cartao_itau_personnalite.py` — ✅ CONCLUÍDA — 2026-05-06

**Pré-requisito:** TASK-IT-01 concluída e recon aprovado.

**Objetivo:** Novo módulo com lógica de parsing, multi-titular e validação.

**Especificação:**

| Item | Valor |
|------|-------|
| `emissor` | `"itau_personnalite"` |
| `X_DIV` | `355` |
| Fingerprint | `["Personnalitê", "ITAU UNIBANCO"]` |
| Descriptografia | via `pdf_decrypt.py` — padrão idêntico ao Santander/Samsung |

**Regex de extração (página 1):**
```python
RE_TITUL = re.compile(r"Titular\s+([A-Z][A-Z\s]+?)(?=\s{2,}|\n|Cartão)")
RE_CART  = re.compile(r"Cartão\s+([\d.X]+)\s+(\w[\w\s]+)")
RE_VENC  = re.compile(r"Vencimento:\s*(\d{2}/\d{2}/\d{4})")
```

**Lógica multi-titular:**
- Rastrear `titular_ativo` e `final_cartao_ativo` por bloco intermediário
- Linha `"Lançamentos no cartão"` → pular (subtotal intermediário, não é lançamento)
- Linha `"Total dos lançamentos atuais"` → capturar para validação

**Seções:**
- `"Lançamentos: compras e saques"` → processar
- `"Compras parceladas - próximas faturas"` → ignorar integralmente
- `"Pagamento efetuado em DD/MM/YYYY"` → capturar como tipo `"Pagamento"`

**Classificação de tipo:**
| Condição | Tipo |
|----------|------|
| `valor > 0` e `parcela_num > 0` | `"Compra parcelada"` |
| `valor > 0` e `parcela_num = 0` | `"Compra à vista"` |
| `valor < 0` e sem parcela | `"Ajuste"` |
| Linha de pagamento efetuado | `"Pagamento"` |

**Validação:**
- Campo: `"Total dos lançamentos atuais"` (texto unido: `"Totaldoslançamentosatuais"`)
- Tolerância: R$ 0,10
- Ausência: logar WARNING, não abortar

**Critério de aceite:**
- Processa PDF de teste sem erro
- Total validado dentro de R$ 0,10
- Lançamentos com `titular`, `final_cartao`, `tipo` corretos por bloco
